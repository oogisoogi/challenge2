# 기능 단위 유저플로우 (Functional User Flows)

### Flow 1: 홈 접속 및 콘서트 목록 조회
사용자가 플랫폼에 접속하여 예약 가능한 콘서트를 식별하고 상태를 확인하는 단계입니다.

*   **1. 입력 (Input)**
    *   사용자가 브라우저를 통해 메인 URL에 접속합니다.
    *   콘서트 목록을 스크롤하거나, 특정 콘서트 카드를 클릭합니다.
*   **2. 처리 (Process)**
    *   **데이터 호출:** 서버로부터 전체 콘서트 메타 데이터(공연명, 일시, 정원, 현재 예약 수)를 요청합니다.
    *   **상태 판별 로직:** `현재 날짜`와 `콘서트 진행일`을 비교합니다.
        *   조건: `현재 날짜` < `콘서트 진행일` (진행일 전날까지만 예약 가능)
    *   **엣지 케이스 대응:**
        *   데이터가 0건일 경우: 빈 상태(Empty State) 데이터를 반환합니다.
        *   날짜 경계: 사용자가 보고 있는 도중 자정이 지나 `예약 마감`으로 날짜가 변경된 경우, 클릭 시점(서버 시간)에 유효성을 재확인합니다.
*   **3. 출력 (Output)**
    *   **UI 렌더링:** 콘서트 카드 리스트를 표시하며, 각 카드에 `예약 인원/정원`을 노출합니다.
    *   **상태 표시:** 예약 마감된 콘서트는 시각적으로 비활성화(Dimmed) 처리합니다.
    *   **이동:** 예약 가능한 카드 클릭 시 **[콘서트 상세]** 페이지로 이동합니다.

---

### Flow 2: 콘서트 상세 정보 확인 및 예약 진입
상세 정보를 검토하고 예약 프로세스를 시작하는 단계입니다.

*   **1. 입력 (Input)**
    *   홈 목록에서 선택한 콘서트 ID로 상세 페이지에 진입합니다.
    *   `예약하기` 버튼을 클릭합니다.
*   **2. 처리 (Process)**
    *   **데이터 호출:** 해당 콘서트의 상세 정보(설명, 일시, 장소 등)를 불러옵니다.
    *   **유효성 재검증 (Double Check):** 버튼 클릭 시점에 서버 시간 기준으로 **예약 가능 기간(진행일 전날)**인지 다시 한번 검증합니다.
    *   **엣지 케이스 대응:**
        *   유효하지 않은 콘서트 ID일 경우: 홈으로 리다이렉트합니다.
        *   체류 중 마감: 상세 페이지를 보는 동안 예약 시간이 종료된 경우, 버튼 클릭을 차단하거나 실패 처리합니다.
*   **3. 출력 (Output)**
    *   **UI 렌더링:** 콘서트 상세 정보를 표시합니다.
    *   **피드백:** 예약 기간이 지난 경우 버튼을 '마감됨'으로 표시하거나, 클릭 시 "예약 기간이 종료되었습니다" 알림을 띄웁니다.
    *   **이동:** 검증 통과 시 **[콘서트 예약 - 좌석 선택]** 페이지로 이동합니다.

---

### Flow 3: 좌석 조회 및 선택
4x20 그리드 UI를 통해 좌석을 선택하고 관리하는 단계입니다.

*   **1. 입력 (Input)**
    *   콘서트 예약 페이지에 진입합니다.
    *   좌석 그리드(4개 구역) 내 특정 셀을 클릭합니다.
    *   우측 사이드바의 선택 목록에서 'X' 버튼을 클릭하거나, 이미 선택된 좌석을 다시 클릭합니다.
    *   `예약하기(정보입력)` 버튼을 클릭합니다.
*   **2. 처리 (Process)**
    *   **초기 로드:** 전체 320개 좌석의 상태(`비어있음` / `예약됨`)를 매핑합니다.
    *   **좌석 선택 로직:**
        *   `비어있음` 클릭 시: 현재 선택된 좌석이 4개 미만이면 `선택됨`으로 변경하고 목록에 추가합니다.
        *   `선택됨` 클릭 시: `비어있음`으로 복구하고 목록에서 제거합니다.
        *   `예약됨` 클릭 시: 반응하지 않습니다.
    *   **사이드바 동기화:** 선택/해제 액션 시 우측의 잔여 좌석 수와 선택 목록을 실시간으로 갱신합니다.
    *   **버튼 활성화:** 선택된 좌석이 1개 이상일 때만 다음 단계 버튼을 활성화합니다.
*   **3. 출력 (Output)**
    *   **시각적 피드백:** 좌석 상태에 따라 색상을 구분합니다(Available/Reserved/Selected).
    *   **경고:** 4개를 초과하여 선택 시도 시 "최대 4개까지만 선택 가능합니다" 경고 메시지를 노출합니다.
    *   **화면 전환:** 버튼 클릭 시 정보 입력 폼(모달/오버레이)을 노출합니다.

### Flow 4: 예약 정보 입력 및 최종 예약(제출)
예약자 정보를 입력하고, 서버의 동시성 제어를 거쳐 예약을 확정하는 핵심 단계입니다.

*   **1. 입력 (Input)**
    *   예약자명, 휴대폰 번호, 비밀번호(4자리)를 입력합니다.
    *   `결제/예약 완료` 버튼을 클릭합니다.
*   **2. 처리 (Process)**
    *   **유효성 검사:** 필수 항목 입력 여부 및 데이터 형식(번호, 자릿수)을 프론트엔드에서 1차 검증합니다.
    *   **동시성 제어 (Transaction):**
        1.  트랜잭션을 시작합니다.
        2.  요청된 좌석 ID들의 현재 DB 상태를 조회합니다(Row Lock).
        3.  **성공 조건:** 모든 좌석이 여전히 `비어있음` 상태이면 $\rightarrow$ 상태를 `예약됨`으로 업데이트하고 예약 정보를 저장합니다.
        4.  **실패 조건:** 하나라도 이미 `예약됨` 상태이면 $\rightarrow$ 트랜잭션을 롤백(Rollback)하고 실패 응답을 반환합니다.
*   **3. 출력 (Output)**
    *   **성공 시:** **[예약 완료]** 페이지로 이동합니다.
    *   **실패 시:** "이미 예약된 좌석입니다." Alert를 노출하고, 좌석 선택 화면으로 복귀시킵니다(이때 좌석 상태를 최신으로 새로고침).

---

### Flow 5: 예약 완료 확인 및 조회/취소
예약 결과를 확인하거나, 추후 개인정보로 예약을 조회하고 취소하는 단계입니다.

*   **1. 입력 (Input)**
    *   (완료 페이지) 예약 완료 후 자동 진입합니다.
    *   (조회 페이지) 메뉴에서 진입하여 휴대폰 번호와 비밀번호를 입력하고 조회합니다.
    *   조회된 내역 리스트에서 `예약 취소` 버튼을 클릭합니다.
*   **2. 처리 (Process)**
    *   **조회:** 입력된 정보(Phone + PW)와 일치하는 예약 내역을 DB에서 검색합니다.
    *   **취소 로직:**
        1.  트랜잭션을 시작합니다.
        2.  해당 예약 건의 상태를 `취소됨`으로 변경합니다.
        3.  연관된 좌석들의 상태를 즉시 `비어있음`으로 원복하여 타 사용자가 예매 가능하도록 만듭니다.
*   **3. 출력 (Output)**
    *   **UI 렌더링:** 예약 내역 리스트(공연명, 좌석, 날짜 등)를 카드 형태로 표시합니다.
    *   **피드백:**
        *   조회 정보 불일치 시: "일치하는 예약 내역이 없습니다." 메시지를 노출합니다.
        *   취소 성공 시: "예약이 정상적으로 취소되었습니다." 메시지를 띄우고 목록을 갱신합니다.
