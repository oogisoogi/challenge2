# 제품 요구사항 정의서 (PRD)
**Project Name:** No-Login Concert Reservation Platform
**Version:** 1.0
**Date:** 2024-05-20

## 1. 제품 개요 (Product Overview)
본 프로젝트는 별도의 회원가입이나 로그인 절차 없이 누구나 쉽고 빠르게 콘서트 티켓을 예매할 수 있는 웹 기반 플랫폼을 구축하는 것을 목표로 합니다. 복잡한 인증 절차를 제거하여 사용자 진입 장벽을 낮추고, 직관적인 좌석 선택 UI를 통해 사용자 경험을 극대화합니다.

*   **핵심 가치:** 간편함(Simplicity), 속도(Speed), 직관성(Visibility)
*   **주요 기능:** 비회원 예매, 좌석 시각화(바둑판), 실시간 잔여 좌석 확인, 예약 조회 및 취소

## 2. 이해관계자 (Stakeholders)
*   **Target Users (End User):** 회원가입의 번거로움 없이 빠르게 원하는 콘서트를 예매하고자 하는 일반 사용자.
*   **Client/Admin:** 콘서트 정보를 제공하고 판매 현황을 모니터링하는 주최 측.
*   **Developers:** 프론트엔드 및 백엔드 로직 구현 담당자.
*   **Designers:** 직관적인 좌석 UI 및 UX 설계 담당자.

## 3. 포함 페이지 (Included Pages)
총 5개의 페이지로 구성됩니다.
1.  **홈 (Home):** 콘서트 목록 및 기본 정보 노출
2.  **콘서트 상세 (Concert Detail):** 상세 정보 확인 및 예매 진입
3.  **콘서트 예약 (Booking):** 좌석 선택 및 예매자 정보 입력 (핵심 기능)
4.  **예약 완료 (Booking Complete):** 결과 확인 및 티켓 정보 안내
5.  **예약 조회 (Lookup):** 개인정보 기반 예매 내역 확인 및 취소

## 4. 사용자 여정 (User Journey)

### Target User Segment
*   **"스피드 예매족" (Guest User):** 특정 가수의 팬으로, 복잡한 절차 없이 빠르게 빈 좌석을 선점하고 결제(예약)까지 끝내고 싶어 하는 사용자.

### User Flow

**Case 1: 신규 예매 (Booking Flow)**
1.  **[홈]** 사용자는 사이트에 접속하여 '예약 가능(진행일 전날)'한 콘서트를 탐색합니다. (정원 대비 예약 인원 확인)
2.  **[콘서트 상세]** 관심 있는 콘서트를 클릭하여 상세 정보를 확인한 후 `예약하기` 버튼을 클릭합니다.
3.  **[콘서트 예약 - 좌석선택]**
    *   4개의 구역(4x20)으로 나누어진 바둑판 UI에서 빈 좌석을 확인합니다.
    *   원하는 좌석을 클릭하여 선택합니다(최대 4개). 우측 사이드바에서 선택 내역을 확인합니다.
    *   `예약하기` 버튼을 누릅니다.
4.  **[콘서트 예약 - 정보입력]** 이름, 휴대폰 번호, 비밀번호(4자리)를 입력하고 최종 제출합니다.
5.  **[예약 완료]** 예약 성공 시 완료 페이지에서 예매 번호와 좌석 정보를 확인합니다.

**Case 2: 예약 확인 및 취소 (Management Flow)**
1.  **[홈]** 상단 네비게이션(또는 버튼)을 통해 `예약 조회` 메뉴로 진입합니다.
2.  **[예약 조회]** 예매 시 사용한 휴대폰 번호와 비밀번호를 입력합니다.
3.  **[예약 조회 - 결과]** 자신의 예매 내역 리스트를 확인합니다.
4.  **[예약 조회 - 취소]** 일정이 변경되어 `예약 취소` 버튼을 누릅니다. 취소가 즉시 반영됩니다.

## 6. 상세 기능 요구사항 (Detailed Requirements)

### P1. 홈 (콘서트 목록)
*   **UI 구성:** 카드 리스트 형태
*   **기능 요구사항:**
    *   모든 콘서트 목록을 조회한다.
    *   **예약 가능 조건:** 콘서트 진행일 **전날**까지만 예약이 가능하다. (당일 및 지난 콘서트는 '마감' 처리)
    *   **현황 표시:** 각 콘서트 카드에 `현재 예약 인원 / 전체 정원` 텍스트를 노출한다.

### P2. 콘서트 상세
*   **UI 구성:** 콘서트 포스터, 설명, 날짜, 장소 등 상세 정보 영역 + 하단/상단 고정 `예약하기` 버튼
*   **기능 요구사항:**
    *   `예약하기` 버튼 클릭 시 P3(콘서트 예약) 페이지로 이동한다.
    *   예약 마감된 콘서트의 경우 버튼을 비활성화하거나 '예약 마감'으로 표시한다.

### P3. 콘서트 예약 (핵심)
이 페이지는 **(1) 좌석 선택 단계**와 **(2) 정보 입력 단계**로 나뉩니다.

#### 3-1. 좌석 선택 UI
*   **구역 배치:** 전체 좌석은 4개의 구역으로 나뉜다. (예: A구역, B구역, C구역, D구역)
*   **좌석 그리드:**
    *   각 구역은 가로 4칸 x 세로 20칸의 바둑판 형태로 구성된다. (총 320좌석)
    *   각 칸(좌석)은 `비어있음(Available)`, `예약됨(Reserved)`, `선택됨(Selected)` 3가지 상태를 가진다.
*   **인터랙션:**
    *   `비어있음` 좌석 클릭 시: `선택됨` 상태로 변경.
    *   `예약됨` 좌석 클릭 시: 반응 없음 (또는 비활성 표시).
    *   `선택됨` 좌석 클릭 시: 선택 해제되어 `비어있음`으로 변경.
    *   **제약 사항:** 한 번에 최대 **4칸**까지만 선택 가능하다. 4칸 초과 선택 시도시 경고 메시지(Toast/Alert) 노출.
*   **우측 사이드바:**
    *   **잔여 좌석:** 전체 예매 가능 좌석 수를 실시간으로 표시한다.
    *   **선택된 좌석 목록:** 사용자가 현재 선택한 좌석 번호(예: A-1, B-4)를 리스트로 보여준다.
    *   **삭제 기능:** 목록 내 각 좌석 옆에 `X` 아이콘을 배치, 클릭 시 해당 좌석 선택이 해제된다.
    *   **Action:** `1칸 이상` 선택되었을 때만 `예약하기(정보입력)` 버튼이 활성화된다.

#### 3-2. 정보 입력 및 제출 로직
*   **전환:** 좌석 선택 후 버튼 클릭 시, 화면 전환 없이 모달이나 오버레이, 혹은 DOM 교체 방식으로 입력 폼이 나타난다.
*   **입력 필드:**
    1.  예약자명 (Text)
    2.  휴대폰 번호 (Number/Text)
    3.  비밀번호 (Number, 4자리 제한)
*   **제출 프로세스 (동시성 처리):**
    1.  사용자가 정보 입력 후 `결제/예약 완료` 버튼을 클릭한다.
    2.  **서버 검증:** 선택한 좌석들이 *현재 시점*에도 여전히 `비어있음` 상태인지 확인한다.
    3.  **실패 시:** 선택한 좌석 중 하나라도 이미 예약된 경우, **"이미 예약된 좌석입니다."** Alert를 띄우고 좌석 선택 단계로 되돌아간다(이미 선점된 좌석은 갱신하여 보여줌).
    4.  **성공 시:** 모든 좌석이 예약 가능하면 데이터베이스에 저장하고 P4(예약 완료) 페이지로 이동한다.

### P4. 예약 완료
*   **UI 구성:** 예약 성공 메시지, 예약 정보 요약(공연명, 날짜, 좌석 정보, 예약자명)
*   **기능 요구사항:**
    *   `예약 취소` 버튼을 제공한다. 클릭 시 즉시 취소 처리 후 홈으로 이동하거나 취소 완료 메시지를 보여준다.
    *   `홈으로 가기` 버튼을 제공한다.

### P5. 예약 조회
*   **UI 구성:** 조회 폼(상단) + 결과 리스트(하단)
*   **기능 요구사항:**
    *   **인증:** 휴대폰 번호와 비밀번호(4자리)를 입력받아 일치하는 예약 내역을 조회한다.
    *   **목록 표시:** 해당 정보로 예약된 모든 건을 카드 리스트 형태로 보여준다. (공연명, 좌석 번호 등)
    *   **취소 기능:** 각 예약 건 또는 좌석별로 `예약 취소` 버튼을 제공한다. 취소 시 해당 좌석은 즉시 `비어있음` 상태로 변경되어 다른 사용자가 예매 가능해져야 한다.
